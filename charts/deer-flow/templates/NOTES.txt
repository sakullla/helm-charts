3. Architecture (path-based routing):
   - /                    → frontend  (Next.js,   port {{ .Values.frontend.service.port }})
   - /api/*, /health, /docs, /redoc, /openapi.json
                          → gateway   (FastAPI,    port {{ .Values.gateway.service.port }})
   - /api/langgraph/*     → langgraph (LangGraph,  port {{ .Values.langgraph.service.port }})
   {{- if .Values.provisioner.enabled }}
   - /api/sandboxes       → provisioner (K8s,      port {{ .Values.provisioner.service.port }})
   {{- end }}

   Note: Ingress and HTTPRoute both route directly to the backend services.
   Path /api/langgraph/* routes to langgraph; all /api/* routes to gateway.

4. Required secrets (set before deploying):
   - TAVILY_API_KEY, JINA_API_KEY      (search APIs)
   - BETTER_AUTH_SECRET                (frontend auth)
   - At least one LLM key: OPENAI_API_KEY, GEMINI_API_KEY, or DEEPSEEK_API_KEY

   Example:
   helm upgrade --install my-deer-flow charts/deer-flow \
     --set secrets.TAVILY_API_KEY=tvly-xxx \
     --set secrets.BETTER_AUTH_SECRET=$(openssl rand -hex 32) \
     --set secrets.OPENAI_API_KEY=sk-xxx

5. Images must be built from https://github.com/sakullla/deer-flow:
   - backend/Dockerfile  → gateway.image.repository + langgraph.image.repository
   - frontend/Dockerfile → frontend.image.repository
   - docker/provisioner/Dockerfile → provisioner.image.repository (if enabled)
